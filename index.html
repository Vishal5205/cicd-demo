End-to-End DevOps CI/CD Pipeline

Professional DevOps Automation Project

Project Overview

This project demonstrates a complete enterprise-style DevOps CI/CD and GitOps workflow where application code is automatically built, analyzed, containerized, and deployed to Kubernetes with zero manual intervention.

The goal of this project was to simulate real production DevOps practices from developer commit to live Kubernetes deployment using modern automation tools.

Technology Stack & Tools (Integration & Automation)

These tools work together to form a fully automated CI/CD pipeline where each stage is integrated for seamless delivery from code commit to production deployment.

Detailed Deployment Flow

  1. Developer pushes application source code to GitHub.
  2. GitHub Webhook automatically triggers Jenkins pipeline.
  3. Jenkins checks out source code using Jenkinsfile.
  4. SonarCloud performs code quality and security scanning.
  5. Jenkins builds Docker container image.
  6. Docker image is pushed to Docker Hub.
  7. Kubernetes deployment manifests reference latest image.
  8. ArgoCD continuously monitors Git repository.
  9. ArgoCD syncs changes into Kubernetes cluster.
  10. Kubernetes performs rolling update of application pods.
  11. Application is exposed externally using NodePort service.
  12. User accesses application via EC2 public IP.
